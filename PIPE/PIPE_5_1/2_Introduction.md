# 1. Preface
## 1.1 Scope of this Revision

PCI Express、SATA、USB、DisplayPort 和融合 I/O 架构（PIPE）的物理层接口旨在支持开发功能等效的 PCI Express、SATA、USB、DisplayPort 和融合 I/O 物理层。此类物理层可作为 独立集成电路（IC）提供，也可作为宏单元集成到 ASIC 设计中。本规范定义了一组必须集成在符合 PIPE 标准的 PHY 中的 PHY 功能，并定义了此类 PHY 与介质访问层 (MAC) 及链路层 ASIC 之间的标准接口。本规范无意定义符合标准的 PHY 芯片或宏单元的内部架构或设计。PIPE 规范的制定旨在允许采用多种实现方法。在可能的情况下，PIPE 规范会引用 PCI Express 基础规范、SATA 3.0 规范、USB 3.1 规范、DisplayPort 1.4 规范或 Converged IO 1.0 
规范，而非重复其内容。若出现冲突，PCI-Express 基础
规范、SATA 3.0 规范、USB 3.1 规范、DisplayPort 1.3 规范以及
Converged IO 1.0 规范应优先于 PIPE 规范
