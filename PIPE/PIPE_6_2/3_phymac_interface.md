# 3. PHY/MAC Interface（PHY/MAC 接口）

本章定义 MAC 与 PHY 间的信号分组、方向与基本时序语义，是后续功能章节的基础。

## 3.1 接口分组

PIPE 接口可分为四大类：

1. **Data Interface（数据接口）**
   - 发送数据（TxData/TxDataK 等）
   - 接收数据（RxData/RxDataK 等）
2. **Command Interface（命令接口）**
   - 模式控制、速率控制、电源状态控制、检测触发
3. **Status Interface（状态接口）**
   - 接收状态、链路状态、错误/异常状态反馈
4. **Message Bus Interface（消息总线接口）**
   - 用于寄存器化配置与状态查询

## 3.2 架构相关性

- **SerDes 与 Original PIPE 共用信号**：用于跨架构保持一致语义。
- **SerDes 专用信号**：用于低引脚、复用时钟/通道控制等。
- **Original PIPE 专用信号**：用于传统并行位宽路径下的控制表达。

## 3.3 时钟与复位要点

- 接口行为受 PCLK / MacCLK 与模式配置共同约束。
- 在复位期间，PHY 与 MAC 必须进入定义好的默认输出状态。
- 模式切换、速率切换期间应遵循“命令先建立、状态后确认”的握手顺序。

## 3.4 功能行为总览

本章信号定义将被后续章节引用，用于描述：

- PCIe/USB/USB4/DisplayPort 的收发与低功耗行为；
- SATA 专用发射与接收行为；
- Message Bus 地址空间映射及读写语义；
- 运行时错误检测、缓冲补偿与回环行为。

## 3.5 工程实现建议

- 推荐在 RTL 中按“命令/状态/数据”拆分模块，便于多协议复用。
- 对状态码和寄存器字段建立统一枚举，减少跨模式歧义。
- 在验证中优先覆盖：复位、模式切换、速率切换、电源状态切换、回环路径。
